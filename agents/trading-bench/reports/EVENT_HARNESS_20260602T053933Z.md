## 1. Why this was built

Every prior event-driven / basket non-price lane (PEAD 2026-05-30, overnight-SPY 2026-05-30) showed *real gross signal direction* but was structurally un-evaluable on the existing single-symbol, isolated-$1000-per-symbol harness: it cannot compute a portfolio-level Sharpe/drawdown, cannot carry N concurrent event positions, and cannot share one cash book across symbols. Both prior reports independently concluded "the next dollar of effort goes into a multi-symbol event-driven shared-cash harness, not more parameter tuning." This is the structural force-multiplier — one infra bet unlocks small-cap PEAD, overnight-basket, and true portfolio cross-sectional at once.

## 2. The harness — `runner/backtest_event.py` (NEW, 33 KB)

Public API (verified on disk):
- `EventContext`, `EventDecision` — the clean strategy interface. A strategy module exposes a `decide_event(ctx) -> EventDecision` hook; `ctx` only ever sees bars with `t <= clock_t`.
- `resolve_event_entries(...)` — maps raw event dates → entry bars. Reaction bar = first bar on/after the event (8-K filing) date; **entry fills at the NEXT bar's close** (never the reaction-day close → no lookahead). Weekend/holiday roll handled; events too recent (no post-bar) or after all bars are dropped.
- `backtest_event(...)` — the engine. Single shared clock (`build_clock`-style) across the whole universe. Each tick: process exits first (stop / take / hold-horizon / window-end liquidation; tie-break stop>take>time), then process new entries firing that bar, all drawing from / returning to **ONE shared cash balance**. Carries up to `max_concurrent` positions; declines new entries when cash is exhausted or the cap is hit (counted, not silently dropped).
- Outputs: portfolio equity curve → `fp_continuous_sharpe` (the only honest Sharpe), portfolio max-DD AND `worst_instrument_dd_pct` (per GATE #5(b)), annualized return on deployed notional, win rate, trade count, total costs, and a **same-path BH-SPY benchmark** (`_bh_spy_return_on_clock`).
- Cost model mirrors the existing harness (round-trip ~4 bps equities); `notional_usd` and the $1000 paper cap respected. No protected file touched.

## 4. The proving lane — small-cap PEAD (`strategies_candidates/pead_event_smallcap/`)

Post-Earnings-Announcement-Drift (Bernard-Thomas 1989), price-reaction proxy, run on the universe where the inattention edge actually lives: **29 hand-picked small/mid-cap US names** across sectors (AAON, ANF, ATKR, BLDR, CALM, CELH, CROX, CRS, CVNA, DECK, DOCN, ELF, FIVN, FSLR, KFRC, MTH, NSSC, PLAB, POWL, RMBS, SHAK, SHLS, SMCI, SPSC, TGLS, TXRH, UFPT, UPWK, WING).

Construction (fixed before the run, NOT tuned to results):
- Event = SEC EDGAR Form 8-K Item 2.02 filing date (free earnings-date source; fetched by `earnings_universe.py`).
- Surprise = (pre-event close → reaction close) ≥ **+3%** (the `pead_lib` default, FIXED).
- Entry = next bar close after the reaction bar. Hold 42 trading bars (~Bernard-Thomas 60-cal-day drift horizon), −5% stop / +10% take, first trigger wins.
- Up to 10 concurrent $100 positions from one shared $1000 book.

## 5. Survivorship caveat (disclosed, and it strengthens the reject)

Universe is hand-picked from *currently-listed* names → **positive survivorship bias**. Reported edge is therefore an **UPPER bound**. The strategy rejects even with that tailwind, so the true (delisting-aware) result is at-best no-better. The bias direction makes the reject MORE robust, not less. Documented in `params.json` and `earnings_universe.py`.

## 6. No-lookahead confirmation

Entry index is strictly `reaction_bar + 1` (pinned by `test_entry_fills_after_reaction_bar` + `test_reaction_and_entry_indices`); `decide_event` receives only `bars[t <= clock_t]` (pinned by `test_decide_event_never_sees_future`). Reaction bar itself (the surprise bar) is never the fill bar.

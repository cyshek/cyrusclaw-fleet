## 1. R3-A — TRUE proportional construction, expressed WITHOUT protected edits

Candidate `strategies_candidates/vol_regime_spy_prop/` (`decide_xsec` + params.json). Modeled the vol-managed sleeve as a **single-name "basket" = {SPY}** on the xsec path:

- `starting_cash = MAX_POSITION = $1000`. Each tick the strategy deploys `target_fraction × $1000` to SPY and leaves `(1−fraction) × $1000` as **idle cash** inside the portfolio NAV. That idle cash is exactly the un-invested sleeve — so portfolio equity = cash + SPY value = a genuinely fractionally-deployed vol-managed position. **This is continuous inverse-vol sizing expressed through the EXISTING `backtest_xsec` idle-cash mechanism — ZERO edits to any protected/evaluator file.**
- `target_fraction = clip(target_vol / realized_vol, 0, 1)` (the literal Moreira-Muir map), or for an implied-vol gauge `clip(SMA_of_gauge / gauge, 0, 1)` (fraction=1 at the trailing norm, scales down proportionally as the gauge rises above it).
- **Resize mechanic** (the backtester adds on buy / frees fully on close, no partial-trim primitive): we CLOSE-then-REBUY at the new size only when `|target − current| > resize_band` — between rebalances we hold (no churn). Each rebalance = 1 close + 1 buy = 2 trades, fits the single-name 4/day cap. This is faithful continuous sizing with a deliberate hysteresis band, NOT the round-2 degenerate "near-always-full-notional" collapse.

**The round-2 implementation limit is GONE.** Average deployment for the top proportional cells is 0.83–0.96 (it genuinely sizes down in vol spikes and re-levers in calm) — verified below.

## 1. Why this harness exists (the blocker)

The prior single-symbol harness (`runner/backtest.py`) allocates an **isolated $1000 per symbol**. For an event-driven strategy that fires N concurrent earnings positions, that:
- inflates capital (N symbols × $1000, not ONE $1000 book),
- produces **no portfolio Sharpe / drawdown** (each symbol is its own silo),
- can't model the real constraint: *shared cash + a concurrency cap*.

`runner/backtest_xsec.py` solved this for *continuous rebalancing* cross-sectional baskets, but PEAD is **discrete event entries with per-position holds/stops**, not a daily-rebalanced basket. So a new harness was required. `backtest_event.py` reuses xsec's no-lookahead discipline and the shared `CostModel`, but owns an event→entry→hold/stop lifecycle on ONE cash book.

## 2. Harness design — `runner/backtest_event.py`

### 2.1 Strategy interface (clean, documented)

A strategy module exposes:

```python
def decide_event(ctx: EventContext) -> EventDecision | None
```

**`EventContext`** (everything the strategy may legally see — no future bars):
| field | meaning |
|---|---|
| `symbol` | ticker |
| `clock_t` | current shared-clock timestamp (the entry-candidate bar) |
| `event_date` | the 8-K / earnings date that triggered this evaluation |
| `bars` | **no-lookahead** history, `bars[-1]` = entry-candidate bar (t == clock_t), `bars[-2]` = reaction/surprise bar. **Every** `b["t"] <= clock_t`. |
| `reaction_bar` | the surprise bar (first bar on/after `event_date`) = `bars[-2]` |
| `last_price` | `bars[-1]["c"]` (the fill reference) |
| `params` | strategy params dict |
| `state` | per-symbol scratch dict (persists across that symbol's events) |

**`EventDecision`**: `enter: bool`, `notional_usd: float`, `reason: str`.

The strategy decides **direction/sizing/whether to enter**. The **harness owns** shared cash, the concurrency cap, the exit policy (hold/stop/take), the cost model, and all no-lookahead enforcement. This keeps signal logic and portfolio mechanics cleanly separated.

### 2.2 Event → entry resolution (`resolve_event_entries`)

For each event date:
- **reaction bar** = first bar with `date >= event_date` (rolls weekends/holidays forward),
- **entry bar** = reaction bar **+ 1** (`require_gap_after_event=True`).

So the fill is **the bar AFTER the surprise bar** — never the surprise close itself. This replicates the PEAD report's entry discipline (enter *after* the reaction). Events whose entry bar would fall past the last bar are **dropped** (too recent). `require_gap_after_event=False` exists for unit tests only.

### 2.3 Single shared clock, exits-before-entries

`build_clock()` = sorted union of all symbols' bar timestamps. At each tick:
1. **Mark** open positions to the current bar.
2. **Process exits** first (free cash), then
3. **Process entries** firing this bar, drawing from the freed shared cash,
4. **Record** portfolio equity = cash + Σ open-position market value.

Exit policy (harness-owned, first trigger wins, tie-break **stop > take > time**):
- `stop_pct` (default −5%), `take_pct` (default +10%), `holding_bars` (default 42 ≈ 60 calendar days), and a **window-end liquidation** for anything still open at the last bar.

Constraints enforced every tick:
- **Shared cash floor** — an entry is declined (`skipped_cash`) if `notional > cash`. Cash never goes negative.
- **Concurrency cap** — `max_concurrent`; excess simultaneous events → `skipped_cap`.
- **One position per symbol** — a second event while already long that symbol → `skipped_strategy`.

### 2.4 Metrics (honest)
- `fp_continuous_sharpe` over the **concatenated per-tick portfolio-equity return series** (the only honest Sharpe per MEMORY.md — NOT median-of-windows).
- `max_drawdown_pct` (portfolio NAV) **and** `worst_instrument_dd_pct` (per GATE #5(b)).
- `ann_return_on_deployed`, `win_rate`, trade count, costs.
- `beats_bh_spy` — buy-and-hold SPY over the **same clock span / same path**.

### 2.5 Cost model
Reuses `runner.backtest.CostModel` (the same one xsec/backtest use). Lane runs `CostModel.for_symbol` → equities `spread_bps≈2` one-way (~4bps round-trip). Spot-check below confirms the spread is applied to fills.

### 4.2 Universe & survivorship (HONEST bias note)
29 hand-picked currently-listed US small/mid-caps across sectors (CROX, DECK, BLDR, RMBS, SMCI, CELH, ELF, ANF, PLAB, FSLR, WING, TXRH, CALM, MTH, SHAK, CVNA, UPWK, FIVN, DOCN, AAON, ATKR, CRS, KFRC, SPSC, NSSC, UFPT, POWL, TGLS, SHLS). This is **small/mid-cap** — the inattention universe where Bernard-Thomas PEAD actually lives, NOT mega-cap.

⚠️ **Survivorship bias = POSITIVE.** This list is **not delisting-aware** — every name still trades today, so failed/delisted small-caps are excluded. That biases results *upward*. **The true edge is ≤ what's reported here.** Since the lane already **REJECTS** on the upper-bound list, a survivorship-clean list would only reject harder — the conclusion is robust to the bias.

### 4.3 Data coverage constraint (transparent)
Alpaca IEX free tier returns ~493 daily bars (**2024-06-12 → 2026-06-01, ~2yr**). Earnings history fetched spans 2004–2026 (1,412 distinct events), but only events **inside the bar window** are tradeable:
- `events(total distinct) = 1412`
- `in_window (genuinely tradeable, reaction bar not clamped to bar[0]) = 241`
- of those 241, **63 fired the +3% surprise** → **63 entries** across **27 of 29 names**.

Pre-window events collapse onto bar[0] and are filtered by the strategy's history guard (`len(bars) < 3`) — verified: **0 trades** have an entry before 2024-07-01, and all 63 trade event-dates fall in `2024-07-31 → 2026-05-27`.

**n = 63 trades — NOT low-n.** Reported honestly; no threshold tuning.

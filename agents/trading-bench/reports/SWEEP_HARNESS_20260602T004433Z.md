## TL;DR

Built `runner/sweep.py` — a reusable parameter/universe sweep harness — plus a
consolidated canonical FP-Sharpe helper `runner/fp_sharpe.py` and 25 tests in
`tests/test_sweep.py`. The harness takes a sweep spec (strategy family + param
grid + optional universes), runs the **cartesian product** through the
**existing** walk-forward evaluators (it does NOT reimplement them), ranks
every cell by the **canonical full-period continuous-span Sharpe**, and
**auto-classifies** each front-door-passing cell as a robust **PLATEAU** or an
overfit **KNIFE-EDGE**. Cost model is always active (asserted). A combinatorial
guard refuses > `cell_cap` (default 500) cells.

**Validation:** ran it on the already-characterized `xsec_ss_momentum_lc20_v2`
family over a 2×2 (cadence × K) grid on the 20-name universe. It **exactly
reproduced the known reject** from `reports/SS_MOMENTUM_MONTHLY_*` — canonical
12-1 monthly K5 = FP-cont Sharpe **−0.11**, all 4 cells REJECT, **zero
plateaus** (basin uniformly fails, not a knife-edge ridge). The harness agrees
with the hand-run numbers to the cent.

## 1. The bottleneck this removes

Every recent swing hand-spawned a subagent that tested ~5 param cells with an
ad-hoc throwaway driver. Each one reinvented the same wheel:
`reports/_lowturn_driver.py`, `reports/_ss_momentum_driver.py`,
`reports/_dispersed_universe_driver.py`, `reports/_ss_momentum_jitter.py`,
`reports/_lowturn_fpsharpe.py`, … Each re-implemented grid iteration, the
FP-Sharpe, the deployed-ann-return, the cost assert, and a by-hand jitter check.

That method has two failure modes: (a) it only ever eyeballs ~5 cells, so a
real plateau three steps away is never found; (b) the load-bearing FP-Sharpe is
copy-pasted 3+ times with a hardcoded `sqrt(252)` — exactly the kind of
duplicated, untested ruler that produces silent drift. The harness fixes both.

## 2. API / usage

```python
from runner.sweep import SweepSpec, run_sweep

spec = SweepSpec(
    family="xsec",                       # "xsec" (basket) or "single"
    decide_fn=decide_xsec,               # candidate's decide_xsec / decide
    base_params=params,                  # params.json dict; cells override keys
    grid={"lookback_bars": [126, 252],   # param_name -> list of values
          "top_k": [3, 5]},
    universes={"lc20": UNIVERSE,         # name -> list[str] OR path to basket
               "wide": "baskets/wide.txt"},  #   file (one/line or JSON list)
    strategy_label="xsec_mom_v2",
    warmup_days=420,                     # primes slow lookbacks (xsec)
    cell_cap=500,                        # combinatorial-explosion guard
)
report = run_sweep(spec)
print(report.to_markdown())              # ranked report + surfaced plateaus
robust = report.plateaus()               # cells worth a human look
```

- **Loader-free:** runs via the `decide_fn` + `params` override path, so a
  candidate never needs to be installed in `strategies/`.
- **`family="single"`** routes to `runner.walk_forward.walk_forward`; universes
  are ignored (symbol comes from params). `family="xsec"` routes to
  `runner.walk_forward_xsec.walk_forward_xsec`.
- **Cartesian product:** `grid` × `universes`. Empty grid = one cell
  (base_params). Errored cells (e.g. `ZeroTradesError`) are recorded with
  `.error`, sorted last, never count as a pass, never anchor a plateau.

## 3. Canonical FP-Sharpe consolidation (a major correctness win — flagged)

`runner/fp_sharpe.py` is now the **single, tested** implementation of the
load-bearing metric:

> **Full-period continuous-span Sharpe** = concatenate every NAMED_WINDOWS
> window's per-tick equity returns into ONE series, annualize with
> `sqrt(bars_per_year(timeframe, is_crypto))` — **√252 equities / √365 crypto**
> per the corrected 2026-05-31 `bars_per_year` convention. A basket is crypto
> only if ALL legs are `/`-symbols (mirrors `backtest_xsec.xsec_is_crypto`).

**This is the number GATE clause (a) binds on.** Median-of-windows Sharpe is a
MIRAGE — it is generally higher and has already caused one bad promotion
(`xsec_momentum_xa`) plus nearly two more. The harness reports FP-cont Sharpe as
the PRIMARY rank key and labels median-window Sharpe explicitly
SECONDARY/generous in every row.

**Throwaway drivers this replaces** (all hand-rolled the same concatenate +
`sqrt(252)` logic, untested):

| Throwaway driver | Duplicated function | Now provided by |
|---|---|---|
| `reports/_lowturn_fpsharpe.py` | `fp_sharpe(agg)` | `fp_sharpe.fp_continuous_sharpe` |
| `reports/_ss_momentum_driver.py` | `fp_continuous_sharpe(agg)` | `fp_sharpe.fp_continuous_sharpe` |
| `reports/_ss_momentum_driver.py` | `deployed_ann_return(agg, dep)` | `sweep.deployed_ann_return_pct` |
| `reports/_lowturn_driver.py` (+ `_dispersed_universe_driver.py`, `_eval_tsmom_xa.py`) | inline FP-Sharpe / ann-return copies | the two helpers above |

Public surface of `fp_sharpe.py`: `equity_curve_returns`,
`concat_window_returns`, `sharpe_from_returns`, `fp_continuous_sharpe`,
`fp_continuous_sharpe_for_agg`, `basket_is_crypto`. Future drivers and the
harness import these instead of re-deriving them. **Pinned with a test against
a hand-computed series** (`test_fp_sharpe_against_hand_computed_series`:
returns `[0.01,-0.005,0.02,0,-0.01]` → Sharpe `+3.95491837`, exact), a no-seam-
bridge test, a crypto-√365 ratio test, and degenerate/blow-up guards.

## 4. Output: the ranked sweep report

Per cell, `to_markdown()` emits: param/universe combo, **FP-cont Sharpe (PRIMARY
rank key)**, median-window Sharpe (labeled generous), `worst_instrument_dd_pct`,
ann-return-on-deployed, round-trip count, front-door verdict
(PASS / REJECT(clauses)), and the robustness tag. Cells are ranked by FP-cont
Sharpe descending. A dedicated **"ROBUST PLATEAUS"** section surfaces only the
plateau cells for human review (or states "none" so a lone knife-edge can never
masquerade as trustworthy). Front-door verdict reuses the same gates the manual
drivers asserted: `passes_fitness_gate_xsec` + Bar A bullet #1
(`bar_a_bullet1_pass`) + `passes_bar_a_5b` (deployed-capital DD). Clause (a)
(FP-cont Sharpe < 1.0) is recorded as a reject reason separately.

## 5. Plateau / knife-edge robustness classifier (the single most important feature)

`classify_robustness(cells, grid)` tags every front-door-passing cell:

- **PLATEAU** — at least one immediate neighbor (a cell differing by exactly
  **±1 grid step on a SINGLE axis**) also passes. Surfaced for human review.
- **KNIFE-EDGE** — every ±1 neighbor fails, OR the cell has no in-grid
  neighbors at all (isolated / off-grid pass). Auto-flagged **NOT robust =
  overfit**.

Neighbor lookup is an integer-coordinate walk over the swept axes, **scoped per
universe** (a pass in `u1` is never a neighbor of a pass in `u2`). This is what
lets us trust a "pass" without a human re-running jitter by hand every time — a
knife-edge pass is auto-quarantined.

Pinned by synthetic-grid tests: isolated center pass → KNIFE_EDGE; contiguous
2×2 block → all PLATEAU; two adjacent passes → PLATEAU; two **diagonal** passes
(differ on two axes) → both KNIFE_EDGE; cross-universe same-cell passes → both
KNIFE_EDGE; degenerate single-cell grid → KNIFE_EDGE.

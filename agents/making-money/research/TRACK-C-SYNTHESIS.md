## 1. THE CONVERGED ANSWER (both tracks agree)

**Distribution splits in two:** *making* content (now free with AI ✅) vs *getting it seen* (platform-gated ❌). Reach happens only 3 ways: **PAY**, build a **LOOP**, or own a **ranked INDEX position**. Volume-broadcast alone = shadowban/deindex.

**What an agent team CAN manufacture:**
- **Product-led loops** (the product carries its own reach: send-to-a-non-user). Only pure flywheel; most defensible; no platform to ban it. Calendly >$3B, Loom ~$975M on this mechanic.
- **Marketplace store-rank** (the store's buyer-intent search IS the audience you lack). Proven cold-start: 30-app portfolio → ~$22k/mo. Store maintains the flywheel.
- **Paid reach** — but a $0–50 budget can't bootstrap it (~$120/day floor to train Meta). Accelerant later, never the bootstrap.

**What it CANNOT:** automated organic broadcast at scale (TikTok/Reddit/LinkedIn burners → shadowban/ban) and thin AI-pSEO (fresh test: 262 pages → 21% indexed → 7 clicks/4wk; AIO halves CTR). These are the seductive lanes that don't survive contact.

**The one uncomfortable truth that dominates everything:** the real wall is **IGNITION, not the loop/channel.** Loops and rankable assets AMPLIFY distribution; they almost never CREATE it from a vacuum. The agent edge is *building + running* the machine; a human still strikes the first spark (hand-seed the first cohort — small, scriptable). Anyone who tells you agents conjure cold-start distribution from nothing is selling something.

**Exit reality (recalibration):** 2025-26 AI "billions" are **acqui-hires** (buying *talent*) — a team-less agent business structurally can't sell that. Realistic exit ceiling = **5–8x ARR roll-up** (very reachable if MRR is real) or a **Base44-style ~$80M defensive PRODUCT buy** (genuine lottery tail). Large MRR is solid + evidenced; $80M is a tail outcome, not engineerable to order. Honest, and Cyrus has it.

## 2. THE WINNING SHAPE (where both tracks point)

> **A product whose distribution is built INTO it — a tool that produces a deliverable the user must SEND to a non-user (row-1 loop) — launched MARKETPLACE-NATIVE (row-2 buyer intent), compounding a DATA MOAT over time for the exit tail.**

Distribution baked into the product, not bolted on via content/social agents can't drive. This is the only configuration where "manufacture distribution with agents" is literally true: a loop is your own product surface (un-bannable), a marketplace is an index that wants sellers.

The two survivors from Track B map cleanly onto the two engines from Track A → run as a **BARBELL**:
- **SPINE (speed + thesis proof + cash):** product-led-loop tool on a marketplace foundation. Engine #2 grafted onto Engine #1. EV base $1–5k / good $10–30k / tail roll-up exit.
- **COMPOUNDING LEG (magnitude + exit tail):** data-moat tool (Engine #3 done right — proprietary data, exact-match intent, NOT thin pages). EV base $0–2k / good $10–40k / tail $100k+/mo → data-buy.

Barbell rationale: until we EMPIRICALLY prove a loop self-propagates, every $10k/mo projection is theory. The spine tests the core thesis fast + cheap + throws off cash; the data-moat leg cooks toward the bigger outcome in the background. We learn if the premise holds BEFORE betting a year.

## 3. THE OPEN DESIGN QUESTION (what to build first)

Not "what shape" (settled) — **which specific vertical deliverable** does the loop produce? Requirements for the loop to fire (hard gates from the research):
1. **Output so good the recipient says "whoa"** and screenshots it — or the loop + brand turn NEGATIVE.
2. **Recipient overlaps the ICP** (they're often also a target user → they convert → send their own). This is the Calendly overlap.
3. **The share is MANDATORY to the core job**, not an optional "refer a friend" (optional share → K≈0 → dead).
4. Agent-buildable substance that's free/credible (verified-keyless APIs where possible).

Leading candidate family (from viral-swing + categories research): **an AI client-facing deliverable a freelancer/agency/SMB SENDS to their prospect to win a deal** (audit/proposal/report) with a "⚡ make your own" footer — recipients are often other marketers/business owners = ICP overlap. One engine re-skins across verticals = a family of swings. The experiments below pick the winner empirically rather than guessing the vertical now.

## 4. THE EXPERIMENTS (running now — empirical thesis test, <$50 total)
Each has a PASS/FAIL so we learn from evidence not vibes. Run in parallel (independent, cheap, agent-run).

- **EXP-2 [highest value] — Product-loop K-factor test.** Build the smallest send-to-a-non-user AI deliverable + "make your own" footer; hand-seed ~20–50 users (ignition); instrument K = exposures/user × conversion-of-exposed. PASS: K trending ≥~0.4 AND recipients say "whoa." FAIL: K≈0 → loop is wishful, redesign/kill. **Directly tests "can agents manufacture a flywheel."**
- **EXP-1 — pSEO indexation + intent probe.** One free data-moat micro-tool on an exact-match domain for a real high-intent query (à la aeochecker.ai → $500 MRR, 80% from domain SEO). Submit sitemap day 1. PASS: >40% indexed AND qualified signups. FAIL: ~20%/~0 clicks → thin pSEO confirmed dead, lean harder on moat/intent. **Tests Engine #3.**
- **EXP-3 — Marketplace cold-start probe.** One-time Chrome dev acct ($5); publish a tiny genuinely-useful free extension as a PROBE; measure organic installs/wk from store search alone. PASS: any non-trivial organic trickle → store supplies buyers. FAIL: flat zero → must seed first cohort (informs Engine #1 launch). **Tests Engine #1's "free traffic" claim.**
- (EXP-4 optional — scripted-face short-form reach probe, if spare cycles. EXP-5 paid-search micro-probe ONLY after a converting asset exists.)

Cash exposure <$50. Output: which engine actually produces reach for US, empirically — then we commit the real build to the winner.

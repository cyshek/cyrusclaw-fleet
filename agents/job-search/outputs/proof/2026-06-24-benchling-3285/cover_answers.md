## What AI tools  are you currently using today and how are you using them?

I lean on AI heavily in my day-to-day as a Product Manager. My most hands-on work is with Copilot Studio, where I built an internal AI agent that restructured our planning workflows. That agent cut planning cycle time by 39% and increased drill capacity by 21%, and building it taught me a lot about where LLM-friendly tooling actually delivers for developers versus where it adds noise. I also migrated our team documentation into an AI-powered semantic search tool, which cut lookup time by 83% by letting people ask questions in plain language instead of hunting through pages.

During my internship I owned developer experience for an AI-driven code generation platform built around intent-based YAML generation. Running demos and onboarding sessions for that drove adoption across 14 teams and saved roughly 37 engineering hours a month, so I think a lot about how agentic and LLM-friendly tooling changes what developers can do, not just how it speeds up existing tasks.

Day to day, I use AI assistants to draft and pressure-test product requirements, summarize user interviews and support signals into themes, and sanity-check API specs and code so I can have credible conversations with engineers faster. The pattern I keep coming back to is using AI to compress the slow, manual parts of discovery and writing so I can spend more time on the judgment calls that actually move the roadmap.

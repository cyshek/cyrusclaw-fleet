## Why are you interested in working at Character AI?

At Microsoft, I built an AI agent that restructured internal planning workflows, cutting cycle time by 39% and increasing capacity by 21%. That project taught me something I keep coming back to: the hardest part of AI product work is not wiring up the model, it is translating a raw capability into an experience that actually changes how someone works or feels. That is exactly the problem Character is solving at a much more ambitious scale, across conversations, characters, and creative tools that people spend hours with every day.

What draws me specifically to Character is the bet that emotionally resonant, long-form AI interaction is its own new medium, not a feature bolted onto something else. Most consumer AI products are optimizing for task completion. Character is optimizing for engagement depth and connection, which requires a completely different kind of product thinking. I want to work on the core experiences that define what that medium feels like, and I think the 0-to-1 work I have done building platformized systems from scratch, under ambiguity and with cross-functional ownership, maps directly to what this team needs.

## [Optional] What is one feature you would be excited to add or improve?

I would want to build persistent character memory that actually earns trust over time. Right now, even the best character interactions tend to feel episodic, each conversation starts mostly fresh. For a platform where emotional resonance and session depth are the core metrics, that is a meaningful ceiling. Users who return to the same character repeatedly are signaling they want a relationship, not a series of one-offs.

The feature I would pursue is a structured, user-visible memory layer tied to each character, where the character can recall and reference past conversations in ways that feel natural rather than mechanical. The tricky product challenge is not the retrieval, it is the transparency and control side. Users need to feel that the character remembers them, not that they are being tracked. That means building the right UI for reviewing and editing what a character knows, and setting clear expectations about what persists. Done well, this would directly increase return session depth and strengthen the emotional connection that differentiates Character from every other AI chat product.

I spent time at Microsoft building semantic search and RAG-backed tooling that made large volumes of context retrievable in a structured way, so I have a practical sense of what the underlying system requires. The PM work here is less about the infrastructure and more about defining exactly what the experience should feel like for the user, which is the part I find most interesting.

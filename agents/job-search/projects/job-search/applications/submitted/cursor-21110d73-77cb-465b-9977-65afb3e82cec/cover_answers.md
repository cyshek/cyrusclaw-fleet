## Please write a short note on a project you're proud of:

At Microsoft, I built Azure's first proactive resilience testing capability from scratch - a rack-level drill program I delivered in 4 months that achieved a 94% recovery rate. What made it hard wasn't the technical design, it was that nothing like it existed internally. I had to define provisioning requirements, design the sandboxing and failover model, and coordinate execution across distributed systems, all under executive visibility and tied to a $1.5B+ enterprise contract. The program surfaced critical hardware defects that would have otherwise gone undetected and established a continuous validation loop that became the template for how we ran drills going forward.

I'm proud of it because it forced me to make real decisions with incomplete information at every stage - architecture, orchestration, stakeholder alignment - and still ship on time. It's also the project where I first built out a proper measurement layer, tracking task completion rate, time-to-value, and cost-per-execution so the team had a clear signal on where to invest. That combination of shipping something new, instrumenting it correctly, and seeing it adopted at scale is what stuck with me.

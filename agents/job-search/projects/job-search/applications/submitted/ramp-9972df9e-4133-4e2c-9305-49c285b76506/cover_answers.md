## Tell us about a time where you successfully worked with engineering, design, and data teams to upgrade an experience. Please be specific on how you championed the user experience?

At Microsoft, I led the build of an internal AI agent for drill planning that required tight coordination across engineering, data, and operational stakeholders. The core user problem was that planning a resilience drill involved a fragmented, manual process - engineers were jumping between tools, manually tracking state, and spending cycles on coordination that had nothing to do with the actual technical work. I started by doing discovery with the people running drills to map exactly where time was getting lost, then translated that into a clear set of requirements for the engineering team focused on removing friction at each handoff point.

On the data side, I worked with analysts to instrument the existing workflow so we could baseline planning cycle time and measure improvement - that gave us a shared north star rather than debating subjective wins. For the experience itself, I pushed hard to keep the interface simple even as the underlying LLM-powered automation got more complex. The goal was that users should feel like the tool just handles the boring parts, not that they're operating a new system. We landed on a 39% reduction in planning cycle time and expanded drill capacity by 21% without adding headcount. Championing the user experience in that context meant consistently translating complexity into simplicity, and being the voice in engineering discussions that kept asking whether the user would actually understand what was happening.

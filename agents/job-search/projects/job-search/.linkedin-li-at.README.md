# How to refresh the LinkedIn li_at cookie (2-min, for Cyrus)

The job-search authed resolver reads the raw li_at value from the sibling file
`.linkedin-li-at` and auto-injects it into the browser context on every run
(wired in `role-discovery/linkedin_authed_resolver.py` -> `_inject_li_at`).

## To grab a fresh li_at (Chrome/Edge, while logged into LinkedIn):
1. Open linkedin.com (logged in).
2. F12 -> Application tab -> Storage -> Cookies -> https://www.linkedin.com
3. Find the row named **li_at**. Copy its **Value** (a long string starting `AQED...`).
4. Paste ONLY that value into `.linkedin-li-at` (no quotes, no `li_at=`, no newline noise).
   `printf '%s' '<VALUE>' > .linkedin-li-at`

That's it. Next resolver run auto-injects it. li_at lives ~1 year but a
session can be invalidated sooner (password change, manual sign-out-all,
LinkedIn security action) -> then it reads as DEAD (redirect-loop / authwall).
